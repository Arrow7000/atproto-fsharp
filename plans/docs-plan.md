# Documentation Restructuring Plan

## Status
Phase 7 (type safety) is complete. Docs are ready to be written against the finalized typed API.

## Current State
- fsdocs tooling is set up and working (generates 1,267 HTML pages of API reference)
- Custom Bluesky-themed CSS at `docs/content/fsdocs-theme.css`
- XML doc comments on all public APIs
- 4 existing thin guides (chat, identity, rich-text, pagination) — need replacing/expanding
- `plans/` and `reference/` already moved out of `docs/` so they don't appear in the HTML site

## Vision
Consumer-oriented docs organized around **what users want to DO**, not internal implementation layers. Each guide shows relevant types, fields, and code examples. Stripe-level clarity is the aspiration.

## Agreed Site Structure

```
docs/
  index.md          # Landing page + install instructions
  quickstart.md     # Hello world in 5 minutes
  guides/
    posts.md        # Creating, reading, deleting posts
    profiles.md     # Fetching, understanding profiles
    feeds.md        # Timelines, author feeds, custom feeds
    social.md       # Like, repost, follow, block
    chat.md         # DMs and conversations
    identity.md     # DID/Handle resolution
    rich-text.md    # Mentions, links, hashtags
    media.md        # Image uploads
    pagination.md   # Cursor-based pagination
  content/
    fsdocs-theme.css  # Already exists
  # API reference auto-generated by fsdocs from XML doc comments
```

## What Each Guide Should Cover
- The domain concept ("what does a post look like")
- Relevant types and their fields
- Which functions/namespaces to use
- Code examples for common operations
- Error handling patterns

## Remaining Non-Content Tasks
- GitHub Actions workflow to deploy docs to GitHub Pages on push to main
- Logo/favicon
- README link to the docs site
